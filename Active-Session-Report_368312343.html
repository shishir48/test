EisenVault-Alfresco : Active Session Report

Created by Sonakshi Agarwal, last modified on Aug 27, 2021

This article is about how to add an active session report on client’s branch

The active session report is specific to the client. Like we need to add manually it in the client’s branch.

For Reference - Below is the link to the commit of active session report in Microtek.

https://bitbucket.org/eisenvault/ev-basecode-5.2/commits/4895c6b09a43ba6144cb29c4e6a8789e5ada651a#chg-ev-repo/src/main/resources/alfresco/module/ev-repo/bootstrap/scripts/concurrent-user/concurrentUsers.js

Mainly two files are client specific in which we need to add the client’s name or URL, else the remaining code is same for all.

  1. ev-repo/src/main/resources/alfresco/module/ev-repo/bootstrap/scripts/concurrent-user/concurrentUsers.js

  2. ev-repo/src/main/resources/alfresco/module/ev-repo/bootstrap/eisenvault-email-templates.acp (make a change in concurrent-user ftl).
